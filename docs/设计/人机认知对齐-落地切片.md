# 人机认知对齐「三痛」落地切片（工程备忘）

对应痛点：**用户不信任**、**多轮偏题**、**出错难修**。理论索引见 [智能体人机认知对齐设计指南v0.3.md](./智能体人机认知对齐设计指南v0.3.md) 第 9～10、15 章。

## 已实现（代码侧）

| 方向 | 说明 |
|------|------|
| 信任与诚实性 | `PromptTemplates.COGNITIVE_ALIGNMENT_TRUTHFULNESS` 注入 Ask/ReAct 等主路径；客户端首启能力/免责说明（localStorage） |
| 多轮 STM | 进程内 `ConversationStmBuffer` + `ChatStmHistoryAssembler`（可选 DST `stateJson` 摘要行）→ `ExecutionContext.history` → `PromptAssembly` 拼「近期对话」 |
| 可纠正 | `ChatRequest.correctionNotes` 送模前拼入；`undoLastStmTurn` 丢弃最近一轮 STM；流式 `StreamErrorMessages` 用户可读短句 |
| 记忆闭环（可选） | `persistUserCorrectionAsMemory` + `jingu3.chat.stm-persist-correction-memory` → 写入 FACT 记忆 |

## 配置项（`application.yml` / `jingu3.chat`）

- `stm-enabled`（默认 true）
- `stm-max-pairs`（默认 5）
- `stm-include-dst-snippet`（默认 true）与 `stm-max-dst-chars`
- `stm-persist-correction-memory`（默认 false）
