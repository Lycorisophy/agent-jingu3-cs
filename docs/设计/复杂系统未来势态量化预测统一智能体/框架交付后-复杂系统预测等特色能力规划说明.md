## 1. 定位（必读）

### 1.1 原路线图不动

- **v0.1～v0.8** 的版本目标、史诗顺序、工序与验收，**仅以** [开发路线图.md](../../计划/开发路线图.md) 及各版本目录 `docs/v0.x/`（PRD、概要/详细设计等）为准。
- **本文不向 v0.6 / v0.7 / v0.8 追加**「复杂系统预测」或 UAP 总报告中的**必交付项**；不把这些特色能力写成框架版本的关闭条件。

### 1.2 框架与特色分离

- 路线图至 **v0.8**（含记忆、技能与工作空间、用户与多租户等）交付完成后，**通用智能体框架**视为**基座就绪**（意图、八行动模式、工具与编排、记忆、技能/沙箱、鉴权与隔离等）。
- **复杂系统未来势态量化预测**及 UAP 总报告中的同类能力，属**基座之上的特色功能线**，在 **v0.8 关闭之后**，以 **[开发路线图](../../计划/开发路线图.md) 史诗 ⑥** 的 **v0.10～v0.12** 为正式交付链；详细设计见 [UAP jingu3 技术适配与落地方案](./复杂系统未来势态量化预测统一智能体jingu3架构的技术适配与落地方案.md)。**v0.9** 为框架横切（非 UAP 主线）。

### 1.3 UAP 总报告的技术路线 = 参考 only

- 总报告中的 **Python / LangGraph / Chroma / Neo4j 示例或推荐栈**，用于说明行业常见做法与能力全景，**不替代**本仓库既定技术选型。
- **本仓库实现**以 [jingu3-开发规范.md](../../规范/jingu3-开发规范.md)、路线图正文及 `docs/v0.x/` 设计为准（例如 Spring Boot、Ollama、Milvus 等与当前工程一致的部分）。

## 2. UAP 特色能力（v0.10～v0.12，方向索引）

以下与 [开发路线图 § v0.10～v0.12](../../计划/开发路线图.md) 及 [UAP jingu3 技术适配与落地方案](./复杂系统未来势态量化预测统一智能体jingu3架构的技术适配与落地方案.md) **§6** 一致；**具体以各版本 PRD 为准**。

| 方向 | 说明 |
| :--- | :--- |
| **v0.10** 工具与侧车 | 动力学 **Tool**、Python 侧车契约、可选联网搜索与 RocketMQ |
| **v0.11** 记忆与领域知识 | 情节记忆 Milvus 策略、`domain_knowledge`、与现有记忆管线协同 |
| **v0.12** 意图策略与闭环 | 内部策略状态机、`prediction_tasks`、客户端仪表盘 MVP |
| 垂直场景与数据 | 电网、供应链等需**单独**数据、评测与合规；可在上述版本内分阶段纳入 |
| 与基座的联动 | 不回溯改写 v0.6～v0.8 的 PRD；在 `docs/v0.10`～`docs/v0.12` 中定义增量 |
