# DeerFlow 上下文工程设计总结

## 概述

DeerFlow 的上下文工程采用**分层注入 + 异步更新**的设计理念，通过记忆系统、上下文注入机制和状态管理三个核心模块，实现智能体在长程对话中的上下文保持。上下文工程的目标是在有限的 token 预算内，为智能体提供最相关的历史信息和当前状态。

### 2.2 记忆上下文注入

```python
def format_memory_for_injection(memory_data: dict, max_tokens: int = 2000) -> str:
    """格式化为注入格式：按 section 组装 + 按置信度排序事实"""
    
    sections = []
    
    # User Context
    if user_data := memory_data.get("user"):
        user_sections = [
            f"Work: {workContext['summary']}",
            f"Personal: {personalContext['summary']}",
            f"Current Focus: {topOfMind['summary']}"
        ]
        sections.append("User Context:\n" + "\n".join(f"- {s}" for s in user_sections))
    
    # History
    if history_data := memory_data.get("history"):
        history_sections = [
            f"Recent: {recentMonths['summary']}",
            f"Earlier: {earlierContext['summary']}"
        ]
        sections.append("History:\n" + "\n".join(f"- {s}" for s in history_sections))
    
    # Facts (sorted by confidence, limited by token budget)
    facts = sorted(memory_data.get("facts", []), key=lambda f: f.get("confidence", 0), reverse=True)
    
    fact_lines = []
    running_tokens = base_tokens
    
    for fact in facts:
        line = f"- [{category} | {confidence:.2f}] {content}"
        line_tokens = count_tokens(line)
        
        if running_tokens + line_tokens <= max_tokens:
            fact_lines.append(line)
            running_tokens += line_tokens
        else:
            break  # 超出预算停止
    
    sections.append("Facts:\n" + "\n".join(fact_lines))
    
    return "\n\n".join(sections)
```

**注入示例**：
```xml
<memory>
User Context:
- Work: Senior Python developer at TechCorp, leading the AI platform team
- Personal: Bilingual (Chinese/English), prefers concise responses
- Current Focus: Building agent frameworks, exploring LLM applications

History:
- Recent: Implemented custom memory system for agent, exploring DeerFlow architecture
- Earlier: Built internal tools for code analysis and documentation generation

Facts:
- [context | 0.95] User works on agent frameworks using Python
- [knowledge | 0.90] Expertise in LangGraph and LangChain
- [preference | 0.85] Prefers detailed technical explanations
</memory>
```

## 三、异步记忆更新机制

### 3.1 记忆更新队列

```python
class MemoryUpdateQueue:
    """带防抖机制的异步记忆更新队列"""
    
    def __init__(self):
        self._queue: list[ConversationContext] = []
        self._debounce_seconds = 30  # 配置值
    
    def add(self, thread_id, messages, agent_name=None):
        # 1. 去重：同一线程的更新合并
        self._queue = [c for c in self._queue if c.thread_id != thread_id]
        self._queue.append(ConversationContext(...))
        
        # 2. 重置防抖定时器
        self._reset_timer(self._debounce_seconds)
    
    def _process_queue(self):
        """防抖到期后批量处理"""
        for context in self._queue:
            MemoryUpdater().update_memory(
                messages=context.messages,
                thread_id=context.thread_id,
                agent_name=context.agent_name
            )
```

### 3.3 消息过滤策略

```python
def _filter_messages_for_memory(messages):
    """过滤规则：
    1. 保留：人类消息（移除 <uploaded_files> 标签）
    2. 保留：无 tool_calls 的 AI 消息（最终回复）
    3. 过滤：工具调用消息（中间步骤）
    4. 过滤：带 tool_calls 的 AI 消息（中间步骤）
    5. 过滤：仅上传文件的空消息
    """
    
    filtered = []
    skip_next_ai = False
    
    for msg in messages:
        if msg.type == "human":
            stripped = strip_upload_block(msg.content)
            if not stripped:  # 空消息
                skip_next_ai = True
                continue
            filtered.append(stripped_msg)
            
        elif msg.type == "ai":
            if not msg.tool_calls:  # 最终回复
                if skip_next_ai:
                    skip_next_ai = False
                    continue
                filtered.append(msg)
    
    return filtered
```

### 3.4 上传文件脱敏

# 防止会话 scoped 的上传信息进入长期记忆
_UPLOAD_SENTENCE_RE = re.compile(
    r"[^.!?]*\b(?:"
    r"upload(?:ed|ing)?(?:\s+\w+){0,3}\s+(?:file|files?|document...)"
    r"|file\s+upload"
    r"|/mnt/user-data/uploads/"
    r"|<uploaded_files>"
    r")[^.!?]*[.!?]?\s*",
    re.IGNORECASE
)

def _strip_upload_mentions_from_memory(memory_data):
    # 1. 从 summaries 中移除上传提及
    for section in ("user", "history"):
        for key, val in section_data.items():
            val["summary"] = _UPLOAD_SENTENCE_RE.sub("", val["summary"])
    
    # 2. 从 facts 中移除上传事实
    memory_data["facts"] = [
        f for f in facts 
        if not _UPLOAD_SENTENCE_RE.search(f.get("content", ""))
    ]

### 4.2 状态合并器

```python
def merge_artifacts(existing: list[str] | None, new: list[str] | None) -> list[str]:
    """工件列表合并 - 去重保序"""
    if existing is None: return new or []
    if new is None: return existing
    # dict.fromkeys 去重且保持顺序
    return list(dict.fromkeys(existing + new))

def merge_viewed_images(existing, new) -> dict[str, ViewedImageData]:
    """图片字典合并 - 新值覆盖旧值"""
    if existing is None: return new or {}
    if new is None: return existing
    # 特殊：空字典 {} 表示清空
    if len(new) == 0: return {}
    return {**existing, **new}
```

### 5.1 Token 预算分配

|------|----------|------|
| 记忆上下文注入 | 2000 tokens | 可配置 max_injection_tokens |
| 系统提示词 | 模型限制内 | 包含角色、技能、子代理等 |
| 技能内容 | 按需加载 | 不在提示词中预填 |

### 5.2 防抖策略

```python
# 配置：30秒防抖
debounce_seconds: int = Field(default=30, ge=1, le=300)

# 队列行为：
# 1. 新消息加入时重置定时器
# 2. 定时器到期后批量处理
# 3. 同一线程的多次更新合并为一次
```

### 5.3 上下文生命周期

```
会话开始:
  └── 加载全局记忆 → 注入系统提示词
  └── 加载代理专属记忆（如有）
  └── 初始化工作空间目录

会话中:
  └── 上传文件 → UploadsMiddleware → 注入 <uploaded_files>
  └── 工具执行 → 更新 artifacts/viewed_images
  └── 对话结束 → MemoryMiddleware → 队列 → 异步更新记忆

会话结束:
  └── 保存最终记忆状态
  └── 清理临时文件（如 outputs）
```

## 总结

DeerFlow 的上下文工程具有以下特点：

1. **分层注入**：静态/半静态/动态三层上下文，按需组合
2. **异步更新**：防抖队列 + LLM 分析，避免同步阻塞
3. **记忆分段**：user/history/facts 多维度组织，支持长程记忆
4. **Token 控制**：tiktoken 精确计算，记忆注入可配置上限
5. **状态合并**：Annotated 类型 + 合并器，自动处理中间件状态更新
6. **上下文隔离**：上传文件信息会话级清除，防止污染长期记忆

上下文工程的核心设计目标是在有限的模型 context 内，为智能体提供：
- **相关性**：最相关的历史信息
- **时效性**：当前会话的最新状态
- **完整性**：关键上下文不遗漏
