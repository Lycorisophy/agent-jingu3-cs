# v0.7 接口文档 — 工作空间 REST（Phase 3）

**基路径**：`/api/v1/workspace`  
**统一响应**：`ApiResult<T>`（与全局约定一致）

**前置条件**

| 配置 | 默认值 | 说明 |
|------|--------|------|
| `jingu3.workspace.enabled` | `true` | 为 `false` 时不注册工作空间 Bean，本组接口不可用 |
| `jingu3.workspace.rest-api-enabled` | `true` | 为 `false` 时不暴露 `WorkspaceController` |

**用户身份**：单用户阶段由 `jingu3.user.id` / `jingu3.user.name` 决定；REST 从 `UserConstants` 取当前用户 ID（与工具侧一致），**请求体无需传 userId**。

---

## 1. 工作空间概览

- **GET** `/api/v1/workspace`
- **说明**：确保磁盘目录与 `workspace` 表元数据存在后返回统计。
- **响应**：`ApiResult<WorkspaceSummaryVo>`

**WorkspaceSummaryVo**

| 字段 | 类型 | 说明 |
|------|------|------|
| workspaceId | string | `workspace` 表主键 |
| userId | string | 用户 ID |
| rootPath | string | 用户工作空间根目录绝对路径 |
| quotaMb | number | 配额（MB），来自元数据行或默认 `default-quota-mb` |
| fileCount | number | 目录下常规文件数量 |
| totalSizeBytes | number | 总字节数 |

---

## 2. 磁盘占用统计

- **GET** `/api/v1/workspace/stats`
- **响应**：`ApiResult<WorkspaceFileStatsVo>`（`quotaMb`、`fileCount`、`totalSizeBytes`，不含 `workspaceId` / `rootPath`）

---

## 3. 重置工作空间

- **POST** `/api/v1/workspace/reset`
- **说明**：清空用户目录下内容，不删除根目录本身；**不删除** `workspace` 表行与执行历史（若需一并清理请使用删除接口或另行运维）。
- **响应**：`ApiResult<void>`（`data` 可为 `null`）

---

## 4. 删除工作空间

- **DELETE** `/api/v1/workspace`
- **说明**：删除磁盘用户目录后，删除 `workspace` 表行（级联删除 `workspace_execution`）。
- **响应**：`ApiResult<void>`

---

## 5. 沙箱执行历史

- **GET** `/api/v1/workspace/executions?limit={limit}`
- **参数**：`limit` 可选；默认 `20`，上限受 `jingu3.workspace.execution-history-list-limit` 约束。
- **前置**：`jingu3.workspace.execution-history-enabled=true`（默认）；否则 **`data` 为空数组**，不落库新执行。
- **响应**：`ApiResult<List<WorkspaceExecutionItemVo>>`

**WorkspaceExecutionItemVo**

| 字段 | 类型 | 说明 |
|------|------|------|
| id | string | 执行记录主键 |
| language | string | 如 `python`、`javascript` |
| runMode | string | `INLINE`（内联代码）或 `FILE`（相对路径脚本） |
| relativePath | string | 文件模式时相对工作空间路径；内联模式多为空 |
| codeHash | string | SHA-256 十六进制（内联为源码摘要；文件模式为 `FILE:lang:path` 摘要） |
| success | boolean | 是否判定成功 |
| exitCode | number | 进程退出码 |
| durationMs | number | 耗时（毫秒） |
| errorType | string | 失败类型，成功时可为空 |
| timedOut | boolean | 是否超时终止 |
| createdAt | string | 创建时间（序列化形式以实际为准） |

**说明**：库中 `stdout_snippet` / `stderr_snippet` 经 `execution-history-snippet-max-chars` 截断；完整输出以沙箱工具实时返回为准。

---

## 6. 错误码

业务失败时 HTTP 状态与 `ApiResult.code` 以全局 `ErrorCode` 为准。工作空间服务将磁盘/IO 类问题封装为 **`ServiceException`**，常见为 **`AG_40001`**（`BAD_REQUEST`）及错误信息中的中文说明（如路径非法、配额不足）。
