# v0.7 接口文档 — 工作空间 REST + 技能市场 + 内置工具目录

**统一响应**：`ApiResult<T>`（与全局约定一致）

---

## 0. 内置工具目录（ToolRegistry）

**基路径**：`/api/v1/tools`

| 配置 | 默认值 | 说明 |
|------|--------|------|
| `jingu3.tool.catalog-api-enabled` | `true` | 为 `false` 时不注册 `ToolController`，本路径 **404** |

- **GET** `/api/v1/tools`
- **说明**：返回 **`ToolRegistry`** 当前已注册的**内置工具**（顺序与启动时收集顺序一致）。每项含 **`id`**、**`description`**（与 Ask/ReAct 提示词内工具说明一致）、**`riskLevel`**（`LOW` / `MEDIUM` / `HIGH`，与 `ToolRiskLevel` 一致；如 `workspace_write_file` 为 `MEDIUM`，`workspace_execute_code` 为 `HIGH`，计算器/读文件等为 `LOW`）。**`jingu3.tool.enabled=false`** 时对话侧不走工具执行，本只读目录仍可暴露（便于客户端展示灰显能力）。
- **响应**：`ApiResult<List<ToolListItemVo>>`

**ToolListItemVo 字段**：`id`、`description`、`riskLevel`。

## 1. 技能市场目录与订阅

**基路径**：`/api/v1/skills`

| 配置 | 默认值 | 说明 |
|------|--------|------|
| `jingu3.skill.api-enabled` | `true` | 为 `false` 时不注册 `SkillController` |

- **GET** `/api/v1/skills`
- **说明**：查询 `skill` 表中 **`is_public = true` 且 `status = 'ACTIVE'`** 的记录，按 `updated_at` 倒序，条数上限 **`jingu3.skill.list-max-size`**（默认 `100`）。**不返回** `storage_path`、`file_size`、`checksum` 等内部字段；技能包下载 URL / MinIO 预签名后续迭代。
- **响应**：`ApiResult<List<SkillListItemVo>>`

- **GET** `/api/v1/skills/subscriptions`
- **说明**：查询当前配置用户（`jingu3.user.id`）在 **`user_skill`** 中 **`status='ACTIVE'`** 且关联 **`skill.status='ACTIVE'`** 的记录，联表返回订阅字段与技能元数据；条数上限同 **`jingu3.skill.list-max-size`**。需存在 **`users`** 外键（单用户种子 `001` 与 [dev_seed.sql](../data/seed/dev_seed.sql) 一致）。
- **响应**：`ApiResult<List<SkillSubscriptionItemVo>>`

- **POST** `/api/v1/skills/subscriptions`
- **说明**：为当前用户订阅技能。请求体 **`skillId`**（技能主键）必填；**`users`** 须存在；**`skill`** 须 **`is_public=true` 且 `status='ACTIVE'`**；若该用户对该技能已有 **`user_skill`** 且 **`status='ACTIVE'`** 则**幂等**（不报错）；若曾存在非 ACTIVE 行则更新为 ACTIVE 并同步版本时间戳；否则插入新行。
- **请求体**：`SubscribeSkillRequest`：`{ "skillId": "<id>" }`
- **响应**：`ApiResult<void>`（`data` 可为 `null`）
- **错误**：用户不存在或技能未公开/不可用 → **`AG_40001`**（`BAD_REQUEST`）；技能主键不存在 → **`AG_40401`**（`NOT_FOUND`）。

- **DELETE** `/api/v1/skills/subscriptions/{skillId}`
- **说明**：按 **`user_skill`** 的 **userId + skillId** 删除订阅行；无匹配行时 **`AG_40401`**（`NOT_FOUND`）。
- **响应**：`ApiResult<void>`

**SkillSubscriptionItemVo**：含 `subscriptionId`、`userId`、`subscriptionStatus`、`localVersion`、`serverVersion`、`lastSyncAt`、`externalSkill`、`externalPath` 及技能侧 `skillId`、`name`、`slug`、`description`、`version`、`category`、`tags`、`triggerWords`、`iconUrl`、`official`、`skillStatus`（不含 `storage_path`）。

- **GET** `/api/v1/skills/{slug}`
- **说明**：按 **slug**（唯一索引）查询同上条件的单条技能；不存在、未公开或非 `ACTIVE` 时返回 **404**，业务码 **`AG_40401`**（`NOT_FOUND`）。
- **响应**：`ApiResult<SkillListItemVo>`

**SkillListItemVo 字段**：`id`、`name`、`slug`、`description`、`version`、`category`、`tags`（JSON 数组字符串）、`triggerWords`（JSON 数组字符串）、`iconUrl`、`official`、`status`、`createdAt`、`updatedAt`。

## 2. 工作空间（Phase 3）

**基路径**：`/api/v1/workspace`

**前置条件**

| 配置 | 默认值 | 说明 |
|------|--------|------|
| `jingu3.workspace.enabled` | `true` | 为 `false` 时不注册工作空间 Bean，本组接口不可用 |
| `jingu3.workspace.rest-api-enabled` | `true` | 为 `false` 时不暴露 `WorkspaceController` |

**用户身份**：单用户阶段由 `jingu3.user.id` / `jingu3.user.name` 决定；工作空间 REST 从 `UserConstants` 取当前用户 ID（与工具侧一致），**请求体无需传 userId**。

### 2.1 工作空间概览

- **GET** `/api/v1/workspace`
- **说明**：确保磁盘目录与 `workspace` 表元数据存在后返回统计。
- **响应**：`ApiResult<WorkspaceSummaryVo>`

**WorkspaceSummaryVo**

| 字段 | 类型 | 说明 |
|------|------|------|
| workspaceId | string | `workspace` 表主键 |
| userId | string | 用户 ID |
| rootPath | string | 用户工作空间根目录绝对路径 |
| quotaMb | number | 配额（MB），来自元数据行或默认 `default-quota-mb` |
| fileCount | number | 目录下常规文件数量 |
| totalSizeBytes | number | 总字节数 |

---

### 2.2 磁盘占用统计

- **GET** `/api/v1/workspace/stats`
- **响应**：`ApiResult<WorkspaceFileStatsVo>`（`quotaMb`、`fileCount`、`totalSizeBytes`，不含 `workspaceId` / `rootPath`）

---

### 2.3 重置工作空间

- **POST** `/api/v1/workspace/reset`
- **说明**：清空用户目录下内容，不删除根目录本身；**不删除** `workspace` 表行与执行历史（若需一并清理请使用删除接口或另行运维）。
- **响应**：`ApiResult<void>`（`data` 可为 `null`）

---

### 2.4 删除工作空间

- **DELETE** `/api/v1/workspace`
- **说明**：删除磁盘用户目录后，删除 `workspace` 表行（级联删除 `workspace_execution`）。
- **响应**：`ApiResult<void>`

---

### 2.5 沙箱执行历史

- **GET** `/api/v1/workspace/executions?limit={limit}`
- **参数**：`limit` 可选；默认 `20`，上限受 `jingu3.workspace.execution-history-list-limit` 约束。
- **前置**：`jingu3.workspace.execution-history-enabled=true`（默认）；否则 **`data` 为空数组**，不落库新执行。
- **响应**：`ApiResult<List<WorkspaceExecutionItemVo>>`

**WorkspaceExecutionItemVo**

| 字段 | 类型 | 说明 |
|------|------|------|
| id | string | 执行记录主键 |
| language | string | 如 `python`、`javascript` |
| runMode | string | `INLINE`（内联代码）或 `FILE`（相对路径脚本） |
| relativePath | string | 文件模式时相对工作空间路径；内联模式多为空 |
| codeHash | string | SHA-256 十六进制（内联为源码摘要；文件模式为 `FILE:lang:path` 摘要） |
| success | boolean | 是否判定成功 |
| exitCode | number | 进程退出码 |
| durationMs | number | 耗时（毫秒） |
| errorType | string | 失败类型，成功时可为空 |
| timedOut | boolean | 是否超时终止 |
| createdAt | string | 创建时间（序列化形式以实际为准） |

**说明**：库中 `stdout_snippet` / `stderr_snippet` 经 `execution-history-snippet-max-chars` 截断；完整输出以沙箱工具实时返回为准。

## 3. 错误码

业务失败时 HTTP 状态与 `ApiResult.code` 以全局 `ErrorCode` 为准。工作空间服务将磁盘/IO 类问题封装为 **`ServiceException`**，常见为 **`AG_40001`**（`BAD_REQUEST`）及错误信息中的中文说明（如路径非法、配额不足）。
