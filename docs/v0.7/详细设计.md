# v0.7 详细设计 — 技能扩展 + Workspace

本文档在 [概要设计.md](./概要设计.md)、[接口文档.md](./接口文档.md) 与源码之间给出**可实现的详细契约**：已落地部分与代码一一对应；**未落地**部分明确为后续迭代边界，避免与 [PRD.md](./PRD.md) 中 HITL / MinIO 承诺混淆。

**权威设计稿**：工作空间行为以 [workspace-design.md](../workspace/workspace-design.md) 为准；技能市场与客户端执行以 [skill-system-design.md](../workspace/skill-system-design.md) 为准。本文侧重 **服务端 Java 分包、数据表、配置键、错误语义与测试锚点**。

| 项 | 约定 |

## 2. Workspace 子系统

### 2.1 分层与主要类

| 层次 | 类型 | 职责 |
|------|------|------|
| 根目录与统计 | `WorkspaceManager` / `DefaultWorkspaceManager` | 解析 `jingu3.workspace.root-dir`（配置键 `root-dir`），确保 `{root}/{userId}/` 存在；提供 `Workspace` / `WorkspaceStats` |
| 文件读写 | `WorkspaceFileService` / `DefaultWorkspaceFileService` | 在用户根下读写列举；配额与单文件大小校验 |
| 路径安全 | `PathValidator` | 规范化相对路径、禁止 `..`、统一分隔符；所有文件类 API 与工具必经 |
| 元数据 | `WorkspaceMetadataService` / `DefaultWorkspaceMetadataService` + `WorkspaceMapper` | `workspace` 表行与磁盘根路径、配额 MB 等 |
| 沙箱 | `SandboxExecutor` / `ProcessSandboxExecutor` | 子进程执行 Python / JavaScript；超时与输出截断 |
| 执行历史 | `WorkspaceExecutionRecorder` / `DefaultWorkspaceExecutionRecorder` + `WorkspaceExecutionMapper` | 可选写入 `workspace_execution` |
| 编排 REST | `WorkspaceManagementService` + `WorkspaceController` | 概览、统计、重置、删除、执行历史列表 |

**包根**：`cn.lysoy.jingu3.workspace`（实体与 Mapper 在 `entity` / `mapper`；工具在 `tool`；沙箱在 `sandbox`；安全在 `security`）。

### 2.2 路径与安全（PathValidator）

- **输入**：相对工作空间根的路径字符串（工具与 REST 共用约定）。
- **禁止**：绝对路径、`..` 段、空段、Windows 保留设备名（按实现为准）。
- **输出**：规范化后的相对路径，供 `DefaultWorkspaceFileService` 与 `ProcessSandboxExecutor` 拼磁盘绝对路径。
- **失败**：非法路径抛出 `ServiceException` + `ErrorCode.BAD_REQUEST`（或统一封装），不向调用方泄露服务器其他目录结构。

### 2.3 配额与单文件大小

- **单文件**：`jingu3.workspace.max-file-size-mb`（默认 10）；超限拒绝写入。
- **总配额**：`default-quota-mb`（默认 1024）；为 **0** 时表示不启用总占用校验（与实现一致）；非零时在 `write` 前将「当前目录已用字节 + 本次写入」与配额比较。
- **元数据**：配额 MB 可来自 `workspace` 表行；缺省回落到配置默认。

### 2.4 沙箱执行（ProcessSandboxExecutor）

- **开关**：`jingu3.workspace.sandbox.enabled`（默认 `false`）；为 `false` 时不注册 `workspace_execute_code` 工具 Bean（与 `@ConditionalOnProperty` 链一致，以实际 Bean 条件为准）。
- **语言**：`python` / `javascript`（可扩展枚举见 `WorkspaceExecutionModes` 或等价常量类）。
- **模式**：内联 `code` 或 `relativePath` 指向工作空间内脚本文件；路径仍经 `PathValidator`。
- **资源**：`max-timeout-seconds`、`max-output-chars`、`max-code-chars`；超时时线程中断策略见实现与单测 `ProcessSandboxExecutorTest`。
- **结果**：`SandboxResult` 含成功标志、退出码、截断输出摘要等；供工具返回文本与执行历史落库。

### 2.5 执行历史（workspace_execution）

- **表**：Flyway **V9**（见 [物化清单](../设计/系统数据存储-物化清单与测试数据.md)）。
- **开关**：`jingu3.workspace.execution-history-enabled`（默认 `true`）；关闭后新执行可不落库或落空策略以代码为准，REST 列表可为空数组。
- **摘要字段**：stdout/stderr 经 `execution-history-snippet-max-chars` 截断后写入；完整输出仅以工具实时返回为准（与接口文档一致）。
- **列表**：`GET /api/v1/workspace/executions?limit=`，`limit` 裁剪至 `execution-history-list-limit`。

### 2.6 REST（WorkspaceController）

- **开关**：`jingu3.workspace.enabled` 与 `jingu3.workspace.rest-api-enabled`；任一关闭则不暴露控制器或不注册 Bean（以 `ConditionalOnProperty` 为准）。
- **端点摘要**：见 [接口文档.md §1](./接口文档.md)；统一 `ApiResult`，错误码与 `ServiceException` 全局处理一致。

### 2.7 内置工具（ToolRegistry）

| toolId | 实现类 | 说明 |
|--------|--------|------|
| `workspace_read_file` | `WorkspaceReadFileTool` | 读文本；大小受 `max-file-size-mb` |
| `workspace_list_files` | `WorkspaceListFilesTool` | 列目录 |
| `workspace_write_file` | `WorkspaceWriteFileTool` | 写文件；配额 + PathValidator |
| `workspace_execute_code` | `WorkspaceExecuteCodeTool` | 委托 `SandboxExecutor`；需沙箱开启 |

工具侧 `userId` 与 REST 一致，从 `UserConstants` 注入，**不在 JSON 参数中信任客户端 userId**。

## 3. 技能市场子系统

### 3.1 只读目录与 slug

- **服务**：`SkillService` / `DefaultSkillService` + `SkillMapper`。
- **目录**：`listPublicCatalog` — `is_public = true` 且 `status = ACTIVE`，按 `updated_at` 倒序，条数 `jingu3.skill.list-max-size`。
- **按 slug**：`getPublicBySlug`；未公开或非 ACTIVE 抛 `NOT_FOUND`（业务码与全局 `ErrorCode` 一致）。
- **VO**：`SkillListItemVo`；**不**包含 `storage_path`、`file_size`、`checksum`。

### 3.2 订阅列表

- **Mapper**：`UserSkillMapper#selectActiveSubscriptionsWithSkill`（`user_skill` JOIN `skill`，双方 ACTIVE）。
- **VO**：`SkillSubscriptionItemVo`（含订阅侧与技能侧展示字段，仍无 `storage_path`）。

### 3.3 订阅写入（POST / DELETE）

- **服务**：`DefaultSkillService#subscribe` / `unsubscribe`（`@Transactional`）。
- **subscribe**：校验 `users` 存在（`UserMapper#countById`）；`skill` 存在且公开且 ACTIVE；已 ACTIVE 则幂等；否则更新或插入 `UserSkillEntity`（主键 UUID，版本与时间戳与实现一致）。
- **unsubscribe**：按 `userId + skillId` 删除；0 行则 `NOT_FOUND`。
- **控制器**：`SkillController`；请求体 `SubscribeSkillRequest`（`skillId` `@NotBlank`）。
- **条件装配**：`jingu3.skill.api-enabled`（默认 `true`）控制 `SkillController` 与对应 `DefaultSkillService` Bean。

### 3.4 与后续 MinIO 的边界

- **当前**：REST 仅元数据；客户端若需包体，须等待预签名或网关下载接口（见 [skill-system-design.md](../workspace/skill-system-design.md)）。
- **表字段**：`storage_path` 等可为后续对象存储路径预留；**不得**在未鉴权下载通道中直接拼接为公开 URL。

## 4. HITL 与工具安全分级（目标态与现状）

**PRD 要求**：写 / 执行类高危能力与 `hitl_approval` 队列联动；低危只读默认放行。

**当前仓库（v0.7 抢跑阶段）**：

- 已通过 **路径隔离 + 配额 + 沙箱超时** 降低误用面；
- **尚未**实现：`request_human_approval` 内置工具、ReAct/Plan 挂起至 `APPROVED` 后续跑、流式 `cardKind=action_approval` 等（见 [BACKLOG-横切增强.md](../v0.5/BACKLOG-横切增强.md)）。

**建议后续详细条目**（供排期，不在本版本强行编码）：

| 等级 | 示例 toolId / 操作 | 目标行为 |
|------|---------------------|----------|
| 低 | `workspace_read_file`、`workspace_list_files` | 默认允许；可审计日志 |
| 中 | `workspace_write_file` | 可配置：直接执行 / 仅草稿 / 强制 HITL |
| 高 | `workspace_execute_code` | 生产默认 HITL 或「二次确认」配置；与 `hitl_approval.payload_json`  schema 对齐 |

## 5. 客户端工具与 JSON Schema

- **范围**：JSON Schema 全覆盖、客户端工具上下行 — **未在本版本服务端闭环**；以 [skill-system-design.md](../workspace/skill-system-design.md) 分阶段为准。**只读内置目录**已提供：`GET /api/v1/tools`（`ToolListItemVo`、`ToolRiskLevel`）。
- **服务端契约**：`Jingu3Tool`（`id()` / `description()` / `execute` / `riskLevel()`）与 `ToolRegistry#buildCatalogMarkdown` 继续为 Ask / ReAct / Plan / Agent Team 专员路由提供目录；`ToolRegistry#buildCatalogList` 供 REST 与客户端展示。

## 6. 错误码与降级（摘要）

| 场景 | 倾向 |
|------|------|
| 路径非法、配额不足、文件过大 | `BAD_REQUEST` + 明确中文信息 |
| 技能不存在 / 未公开 / 订阅不存在 | `NOT_FOUND` 或 `BAD_REQUEST`（与 `DefaultSkillService` 实现一致） |
| 沙箱超时 / 进程失败 | 工具返回结构化失败说明；执行历史 `success=false` |
