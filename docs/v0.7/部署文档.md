# v0.7 部署文档 — 工作空间 + 技能市场（只读）

## 1. 与通用部署的关系

构建、JDK、CI 与 [v0.6 部署文档](../v0.6/部署文档.md) / [v0.2 部署文档](../v0.2/部署文档.md) 相同。主配置见 [`application.yml`](../../agent-jingu3-cs-server/src/main/resources/application.yml)。

## 2. 数据源与迁移

- Flyway 脚本源 [`docs/data/migration`](../../docs/data/migration)，构建时复制到 `classpath:db/migration`。
- **V6** 含 **`skill`** / **`skill_version`** / **`user_skill`**（与身份会话同脚本）；**V9**（[`V9__workspace_phase3.sql`](../../docs/data/migration/V9__workspace_phase3.sql)）创建 **`workspace`**、**`workspace_execution`**。本地 H2 与生产 MySQL 均通过 Flyway 升级，勿用手工 `ddl-auto` 替代业务表结构。

## 3. 技能市场只读 API（jingu3.skill）

| 键 | 说明 |
|----|------|
| `api-enabled` | 是否暴露 `GET /api/v1/skills` |
| `list-max-size` | 公开技能列表最大条数 |

依赖 Flyway **V6** 已创建的 **`skill`** 表；无数据时返回空列表。

## 4. 工作空间相关配置（jingu3.workspace）

| 键 | 说明 |
|----|------|
| `enabled` | 是否启用工作空间（管理器、文件服务、工具注册）；`false` 时无 Phase 3 REST |
| `root-dir` | 工作空间物理根；下按 `userId` 分子目录 |
| `max-file-size-mb` | 单文件读写上限 |
| `default-quota-mb` | 默认配额（MB）；**`0` 表示不校验写入配额** |
| `sandbox.enabled` | 是否启用进程沙箱与 `workspace_execute_code` 工具 |
| `sandbox.*` | 超时、输出/代码长度、`python-command` / `node-command` |
| `rest-api-enabled` | 是否暴露 `/api/v1/workspace/**` |
| `execution-history-enabled` | 沙箱执行是否写入 `workspace_execution` |
| `execution-history-snippet-max-chars` | 执行历史 stdout/stderr 落库单字段最大字符数 |
| `execution-history-list-limit` | REST 执行历史列表条数上限 |

## 5. 验收建议

1. **技能列表**：`GET /api/v1/skills` 返回 200，`data` 为数组（无公开技能时为空）；可手动插入 `skill` 行或执行 [`dev_seed.sql`](../data/seed/dev_seed.sql) 验证。
2. **工作空间 REST**：`GET /api/v1/workspace` 返回 200，`data.workspaceId` 非空，`rootPath` 指向可写目录。
3. **配额**：将 `default-quota-mb` 设为较小正数，连续写入超过配额应失败（`AG_40001` 等）。
4. **沙箱**：`sandbox.enabled=true` 且本机 `python`/`node` 可用时，通过对话或工具调用 `workspace_execute_code`；随后 `GET /api/v1/workspace/executions` 有记录（若 `execution-history-enabled=true`）。
5. **删除**：`DELETE /api/v1/workspace` 后磁盘用户目录与 DB 中对应行被清理（执行历史随 `workspace` 级联删除）。
