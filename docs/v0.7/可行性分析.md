# v0.7 可行性分析 — 技能扩展 + Workspace

## 1. 结论摘要

| 主题 | 结论 | 说明 |
|------|------|------|
| 工作空间 Phase 1～3 | **可行，已落地** | 进程沙箱为本地 `ProcessBuilder` MVP；配额为目录扫描估算；执行历史落 MySQL（Flyway V9） |
| 技能市场元数据只读 | **可行，已落地** | 复用 Flyway V6 `skill` / `user_skill`；无对象存储依赖即可列表/详情/订阅查询 |
| 技能包下载（MinIO 预签名等） | **可行，未在本迭代实现** | 需对象存储配置、密钥与权限模型；与 [skill-system-design.md](../workspace/skill-system-design.md) 对齐后单独立项 |
| JSON Schema 全覆盖工具描述 | **部分可行** | 依赖 ToolRegistry 与工具元数据演进；可与 v0.3 工具链分阶段合并 |
| 安全分级 + HITL 高危联动 | **架构可行、工程未闭合** | [详细设计.md](./详细设计.md) 第 4 节已区分目标态与当前实现；与 `hitl_approval`、工具 `toolId` 的硬联动仍待 backlog 收口；PRD 已列风险表 |

## 2. 技术前提（已具备）

- **单用户种子**：`jingu3.user.id=001` 与 [开发路线图](../计划/开发路线图.md) 一致；技能订阅 REST 与 `users` 外键一致。
- **数据库**：H2（开发）/ MySQL（生产）+ Flyway；工作空间表 V9、技能表 V6。
- **统一 API**：`ApiResult` + `ErrorCode`；工作空间 IO 异常映射为 `ServiceException`（BAD_REQUEST）等。

## 3. 风险与缓解（摘录）

- **沙箱安全边界**：仅进程 + 超时 + 输出截断；生产应叠加运维策略（设计稿 §5）；与 HITL 协同见 [PRD.md](./PRD.md) §2。
- **配额性能**：`writeFile` 前全目录统计，大目录时可能偏慢；后续可缓存或异步（Phase 3+ 增强）。
