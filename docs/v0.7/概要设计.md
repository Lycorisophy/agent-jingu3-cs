# v0.7 概要设计 — 技能扩展 + Workspace

## 1. 范围与边界

本版本在 **单用户种子 `001`** 前提下交付：

- **Workspace**：隔离目录、`WorkspaceFileService`、内置工具、可选进程沙箱、配额、执行历史、管理类 REST（详见 [workspace-design.md](../workspace/workspace-design.md)）。
- **技能**：市场**元数据**只读（目录、按 slug、当前用户订阅联表）；技能包**下载 URL / MinIO** 见 [skill-system-design.md](../workspace/skill-system-design.md)，**不在本概要实现清单内**。

## 2. 逻辑架构（摘录）

```
ToolRegistry ← workspace_* 工具 / 其他 Jingu3Tool Bean
WorkspaceManager ← DefaultWorkspaceManager（磁盘根 + 每用户子目录）
WorkspaceFileService ← PathValidator
ProcessSandboxExecutor（可选）← workspace_execute_code
技能 REST ← SkillController → SkillService / Mapper（MySQL skill / user_skill）
```

## 3. 数据与迁移

| 脚本 | 表 |
|------|-----|
| V6 | `skill`、`skill_version`、`user_skill`（及身份会话相关表） |
| V9 | `workspace`、`workspace_execution` |

权威清单见 [系统数据存储-物化清单与测试数据.md](../设计/系统数据存储-物化清单与测试数据.md)。

## 4. 接口契约

HTTP 与 VO 以 **[接口文档.md](./接口文档.md)** 与 **[docs/v0.5/openapi.yaml](../v0.5/openapi.yaml)** 为准；实现类：`WorkspaceController`、`SkillController`。

## 5. 配置命名空间（摘要）

| 前缀 | 用途 |
|------|------|
| `jingu3.workspace.*` | 工作空间根目录、配额、沙箱、REST、执行历史 |
| `jingu3.skill.*` | 技能 REST 开关与列表条数上限 |
