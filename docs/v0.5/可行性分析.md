# v0.5 可行性分析

## 1. 依赖

- v0.3～v0.4 引擎稳定；本版本偏重基础设施与 REST。
- 用户系统仍为单用户 `001` 时，审批人可为配置账号或同用户（JWT 在 v0.8 强化）。

## 2. 风险

| 风险 | 缓解 |
|------|------|
| 范围膨胀 | PRD 明确 Agent Team 仅「边界内增强」 |
| 与 Cron 草案不一致 | 详细设计引用并迭代 [Cron与HITL模块设计草案.md](../设计/Cron与HITL模块设计草案.md) |

## 3. 结论

建议分模块并行：先 HITL 队列 + State 表（共享存储），再 Cron 调度；Agent Team 可最后合入。
