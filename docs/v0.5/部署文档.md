# v0.5 部署文档 — 横切能力

## 1. 环境与构建

与 [v0.2 部署文档](../v0.2/部署文档.md) 相同：JDK 17+、Maven；主配置 [application.yml](../../agent-jingu3-cs-server/src/main/resources/application.yml)。

## 2. 数据源与迁移

- **开发默认**：H2 内存库 + **Flyway**（`classpath:db/migration`）。启动时依次执行 `V1__scheduled_task`、`V2__hitl_approval`、`V3__dialogue_state`。
- **生产**：请覆盖 `spring.datasource.*` 为 MySQL 等持久库；保持 `spring.jpa.hibernate.ddl-auto=validate` 与 Flyway 启用，避免仅依赖 H2。

## 3. 配置项摘要

| 键 | 说明 |
|----|------|
| `jingu3.cron.scheduler-enabled` | 是否轮询执行到期 Cron 任务 |
| `jingu3.cron.poll-interval-ms` | Cron 轮询间隔（毫秒） |
| `jingu3.engine.agent-team.max-specialist-rounds` | Agent Team 子 Agent LLM 轮数（之后还有一次合成调用） |
| `jingu3.user.id` / `jingu3.user.name` | 单用户种子；HITL `resolver_user_id` 等与此一致 |

## 4. 验收操作建议

1. **Cron**：`POST /api/v1/cron/tasks` 创建过去时间的 `nextRunAt`，观察日志或 DB 中 `last_status`。
2. **HITL**：`POST /api/v1/hitl/approvals` → `GET .../pending` → `POST .../approve` 或 `reject`。
3. **DST**：`PATCH /api/v1/dst/{conversationId}` 写入 JSON → `GET` 校验 → `POST .../confirm` 校验 `revision` 递增。
4. **Agent Team**：`POST /api/v1/chat`，body 中 `mode`=`AGENT_TEAM`，核对回复中含「合成答复」与子 Agent 轨迹（非流式）；流式核对 `specialist_1`…与 `synthesize` 步。
