# v0.5 接口文档 — 横切 REST（`/api/v1`）

机器可读摘要见同目录 [openapi.yaml](./openapi.yaml)。

统一响应体为 `ApiResult<T>`：`success`、`code`、`message`、`data`、`timestamp`。下文仅列本版本新增或强相关的服务端路径。

## 1. Cron（已实现）

| 方法 | 路径 | 说明 |
|------|------|------|
| POST | `/api/v1/cron/tasks` | 创建定时任务（单次执行 MVP） |
| GET | `/api/v1/cron/tasks` | 列出当前配置用户下的任务 |

请求/响应字段见代码：`CreateScheduledTaskRequest`、`ScheduledTaskVo`。

## 2. HITL（已实现）

| 方法 | 路径 | 说明 |
|------|------|------|
| POST | `/api/v1/hitl/approvals` | 创建 `PENDING` 审批单 |
| GET | `/api/v1/hitl/pending?conversationId=` | 按会话列出待审批 |
| POST | `/api/v1/hitl/{id}/approve` | 批准 |
| POST | `/api/v1/hitl/{id}/reject` | 拒绝 |

请求体（创建）：`CreateHitlApprovalRequest` — `conversationId`、`runId`（可选）、`payloadJson`（字符串，常为 JSON 文本）。

## 3. DST / 对话状态（已实现，占位）

| 方法 | 路径 | 说明 |
|------|------|------|
| GET | `/api/v1/dst/{conversationId}` | 读取该会话状态；无记录时 404 |
| PATCH | `/api/v1/dst/{conversationId}` | 写入/更新 `state_json`，`revision`+1；可传 `expectedRevision` 乐观锁 |
| POST | `/api/v1/dst/{conversationId}/confirm` | 用户定稿占位：`revision`+1 |

## 4. 对话主链路（既有）

| 方法 | 路径 | 说明 |
|------|------|------|
| POST | `/api/v1/chat` | 同步对话；`mode=AGENT_TEAM` 时走多轮子 Agent + 合成 |
| POST | `/api/v1/chat/stream` | SSE 流式 |

Agent Team 行为受 `jingu3.engine.agent-team.max-specialist-rounds` 约束；流式中 `step` 标签含 `leader`、`specialist_{n}`、`synthesize`。
