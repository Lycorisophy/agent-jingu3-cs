# v0.5 验收清单（对照 PRD AC）

执行方式：本地或测试环境启动 `agent-jingu3-cs-server` 后，按下列步骤勾选。自动化回归以仓库根目录 [`.github/workflows/ci.yml`](../../.github/workflows/ci.yml) 中 **`mvn test`**（快速）与 **`mvn verify`**（含 PMD/SpotBugs）为准。

## AC-1 Cron

- [ ] `POST /api/v1/cron/tasks` 创建任务，`nextRunAt` 为已过去时间，`payload.message` 非空。
- [ ] 等待轮询（或调小 `jingu3.cron.poll-interval-ms`）后，任务 `enabled=false` 且 `lastStatus` 为 `SUCCESS` 或 `FAILED`（视 Ollama 是否可用）。
- [ ] 重启进程后，`GET /api/v1/cron/tasks` 仍能列出历史任务（**持久库**下成立；默认 H2 内存进程重启会丢数据，属预期，生产需外置数据源）。

## AC-2 HITL

- [ ] `POST /api/v1/hitl/approvals`，body 含 `conversationId`、`payloadJson`。
- [ ] `GET /api/v1/hitl/pending?conversationId=...` 可见 `PENDING`。
- [ ] `POST /api/v1/hitl/{id}/approve` 或 `reject`，再查 pending 列表不含该条（或状态已终态）。

## AC-3 DST

- [ ] `PATCH /api/v1/dst/{conversationId}` 写入 `stateJson`（合法 JSON 字符串）。
- [ ] `GET /api/v1/dst/{conversationId}` 返回一致内容与递增的 `revision`。
- [ ] `POST /api/v1/dst/{conversationId}/confirm` 后 `revision` 再次递增。

## AC-4 Agent Team

- [ ] `POST /api/v1/chat`，`mode`=`AGENT_TEAM`，`message` 任意；响应 `reply` 含 **「合成答复」** 及子 Agent 轮次轨迹。
- [ ] `POST /api/v1/chat/stream` 同参数；流式事件中可见 `leader`、`specialist_1`（及更多轮若配置大于 1）、`synthesize` 相关步。

## 生产数据源（发布前）

- [ ] 已配置非内存 `spring.datasource`，且 Flyway 基线含 `V1`～`V3` 迁移。
