## 2. Plan 子任务与工具

### 2.1 策略选项（实现时选一种为主）

- **A**：每个子任务一次 LLM 调用，输出结构化 `{ "useTool": true/false, ... }`，再分支。
- **B**：启发式关键词 + LLM 兜底决定是否调用默认工具集。

详细算法与提示词见实现阶段；须避免子任务无限递归调用工具。

### 2.2 与 Replanner 关系

- 工具执行抛错时，现有 Replanner 可将错误信息纳入重规划；本版本明确 **将工具错误摘要** 传入 `buildReplannerPrompt`。

## 3. Workflow 节点模型

### 3.1 JSON 扩展示例（草案）

```json
{
  "id": "demo-wf-tool",
  "nodes": [
    { "id": "n1", "type": "LLM", "instruction": "概括用户需求" },
    { "id": "n2", "type": "TOOL", "toolId": "calculator", "instruction": "对下列表达式求值：{{prev}}" }
  ]
}
```

- `type` 缺省为 `LLM`，兼容旧文件。
- TOOL 节点：`toolId` 必填；`instruction` 或模板用于生成工具参数（由详细实现定义占位符与 `accumulated` 注入规则）。

### 3.2 执行顺序

- 仍按数组顺序；`accumulated` 语义：TOOL 节点后将工具输出字符串并入上下文，供下一节点使用。
