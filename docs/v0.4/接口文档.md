# v0.4 接口文档

在 [v0.2 接口文档](../v0.2/接口文档.md) / [v0.3 接口文档](../v0.3/接口文档.md) 基础上，**HTTP 与 `ChatRequest` 不变**。本版仅补充 **Plan / Workflow 行为与流式事件**。

## 1. 请求（无变更）

- `mode=PLAN_AND_EXECUTE`：行为与 v0.2 相同；v0.4 起子任务执行路径在 `jingu3.tool.enabled=true` 时可能增加**一轮路由 LLM 调用**并可能产生内置工具调用。
- `mode=WORKFLOW` + `workflowId`：仍指向 classpath `workflows/*.json` 中的定义 id。

## 2. 工作流 JSON 节点（服务端加载）

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| id | string | 建议 | 节点标识，流式 `STEP` 标签使用 |
| type | string | 否 | `LLM` 或 `TOOL`；缺省视为 `LLM` |
| toolId | string | TOOL 时必填 | 与 v0.3 `ToolRegistry` 一致，如 `calculator`、`utc_time` |
| instruction | string | 否 | LLM：节点指令；TOOL：传给工具的**单字符串**参数 |

示例定义见仓库 [`agent-jingu3-cs-server/src/main/resources/workflows/demo-wf-tool.json`](../../agent-jingu3-cs-server/src/main/resources/workflows/demo-wf-tool.json)。

## 3. 流式 `StreamEvent`

- **Plan-and-Execute**：步 1 为计划 `BLOCK`；后续每子任务一步；若某子任务调用工具，在该子任务 `BLOCK` 前可出现 **`TOOL_RESULT`**（与 v0.3 Ask 同结构）。
- **Workflow**：每节点一步 `STEP_BEGIN` / `BLOCK` / `STEP_END`；`TOOL` 节点可能在当步 `BLOCK` 前出现 **`TOOL_RESULT`**。

## 4. 配置

| 键 | 说明 |
|----|------|
| `jingu3.tool.enabled` | `false` 时 Plan 子任务不再做工具路由；Workflow 的 `TOOL` 节点跳过执行并写入占位说明 |
