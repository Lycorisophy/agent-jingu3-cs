# v0.4 可行性分析

## 1. 依赖

- **强依赖 v0.3**：[`ToolRegistry`](../../agent-jingu3-cs-server/src/main/java/cn/lysoy/jingu3/tool/ToolRegistry.java)、[`ToolRoutingParser`](../../agent-jingu3-cs-server/src/main/java/cn/lysoy/jingu3/tool/ToolRoutingParser.java) 与 `jingu3.tool.enabled`；本版本不重复实现注册表。

## 2. 可行性

- Workflow JSON 扩展字段对 Jackson 反序列化友好；旧文件无 `type` 时默认为 LLM，兼容性好。
- Plan 子任务每步在工具开启时**增加 1 次路由用 `generate`**，成本可接受；解析失败回落与 Ask 一致（直走子任务 LLM）。
- Replanner 仍接收异常信息；工具失败若走「失败汇总」提示词则不抛异常，不触发重规划（与 v0.4 详细设计一致）。

## 3. 结论

可实施；并行网关、Flowable 留待后续以降低 v0.4 风险。
