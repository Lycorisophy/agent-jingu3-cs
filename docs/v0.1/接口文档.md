# v0.1 接口文档

## 1. 概述

- **Base URL**：`http://localhost:8080`（默认 Spring Boot 端口，可按配置修改）
- **内容类型**：`application/json`

## 2. 统一响应体 `ApiResult<T>`

所有业务接口（含成功与失败）均使用同一外层结构：

| 字段 | 类型 | 说明 |
|------|------|------|
| success | boolean | 是否成功 |
| code | string | 业务码，成功为 `0`；失败见下表 |
| message | string | 说明文案 |
| data | object / null | 成功时为业务负载；失败时一般为 `null` |
| timestamp | number | 毫秒时间戳 |

### 2.1 常见业务码（`ErrorCode`）

| code | HTTP 状态 | 说明 |
|------|-------------|------|
| 0 | 200 | 成功 |
| AG_40001 | 400 | 请求参数错误（如非法枚举） |
| AG_40002 | 400 | 参数校验失败 |
| AG_50001 | 500 | 服务内部错误 |

业务代码可抛出 `ServiceException` 携带上述枚举；客户端以 `code` 与 HTTP 状态共同判断。

## 3. 健康检查

| 方法 | 路径 | 说明 |
|------|------|------|
| GET | `/actuator/health` | 存活与健康状态（不包在 ApiResult 内，为 Actuator 默认格式） |

## 4. 聊天

### `POST /api/v1/chat`

**请求体（DTO）**

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| message | string | 是 | 用户消息 |
| mode | string | 否 | 显式行动模式，枚举名如 `ASK`、`REACT`、`PLAN_AND_EXECUTE` 等 |
| conversationId | string | 否 | 会话 ID，缺省为 `default` |

**成功时 `data` 类型（VO：`ChatVo`）**

| 字段 | 类型 | 说明 |
|------|------|------|
| userId | string | 配置项 `jingu3.user.id`（默认 `001`） |
| username | string | 默认 `user` |
| reply | string | 模型或 Stub 返回文本 |
| actionMode | string | 实际选用的行动模式枚举名 |
| routingSource | string | `CLIENT_EXPLICIT` / `RULE` / `MODEL` |

**示例**

```http
POST /api/v1/chat
Content-Type: application/json

{
  "message": "你好",
  "mode": "ASK"
}
```

```json
{
  "success": true,
  "code": "0",
  "message": "成功",
  "data": {
    "userId": "001",
    "username": "user",
    "reply": "...",
    "actionMode": "ASK",
    "routingSource": "CLIENT_EXPLICIT"
  },
  "timestamp": 1710000000000
}
```

**失败示例（非法 `mode`）**

```json
{
  "success": false,
  "code": "AG_40001",
  "message": "mode 不能为空",
  "data": null,
  "timestamp": 1710000000000
}
```

HTTP 状态与 `code` 对齐（本例为 400）。

## 5. 错误处理说明

- 参数校验失败、非法参数、业务 `ServiceException`、未捕获异常均由 **`ApiExceptionHandler`** 转为 `ApiResult`，不向客户端返回堆栈详情。
