# v0.1 可行性分析

## 1. 技术可行性

- **Spring Boot 3.x + Java 17**：成熟，与团队栈一致。
- **Ollama 本地推理**：通过 HTTP API；采用 **LangChain4j Ollama** 封装 `ChatLanguageModel`，降低接入成本。
- **三源路由**：客户端参数由 DTO 解析；规则可用内存 Map + 关键词；模型路由用短提示词分类，失败时降级为 `REACT`（与路线图驾驭一致）。

## 2. 依赖与资源

- 开发者本机需安装 Ollama 并拉取模型：`gnremy/qwen3.5-abliterated:35b-a3b`（主模型）。若暂不可用，可通过配置指向其他本地模型做联调。
- 本版本无强制 GPU；大模型推理耗时依赖本机。

## 3. 风险与缓解

| 风险 | 缓解 |
|------|------|
| 模型名称与本地 Ollama 不一致 | `application-dev.yml` 可覆盖；文档注明 |
| 意图分类提示词不稳定 | 解析失败则降级默认模式；日志记录原始输出 |
| 范围蔓延 | PRD 非目标锁定；Stub 模式统一返回说明字符串 |

## 4. 结论

可行，建议按概要设计→详细设计→实现→单测→部署文档顺序推进。
