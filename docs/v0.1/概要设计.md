# v0.1 概要设计

## 1. 系统边界

```
[HTTP Client] --> [ChatController] --> [IntentRouter] --> [ModeRegistry] --> [ActionModeHandler]
                         |                    |
                         +--------------------+--> [PromptAssembly / 最小上下文]
```

- 外部：HTTP JSON API（本版本）。
- 内部：路由决策 → 模式处理器 → Ollama（Ask/ReAct）；Stub 不调用 Ollama。

## 2. 模块划分

| 模块 | 职责 |
|------|------|
| `config` | Ollama `ChatLanguageModel`、用户常量 |
| `engine` | `ActionMode` 枚举、`ModeRegistry`、`ExecutionContext` |
| `engine.routing` | `IntentRouter`（三源优先级） |
| `engine.mode` | 各模式 `ActionModeHandler` 实现 |
| `prompt` | 基础提示词拼装 |
| `web` | REST DTO 与 Controller |

## 3. 与三大系统/工程映射

- `SYS:意图路由` + `SYS:行动模式`：路由 + 注册表 + 处理器。
- `ENG:提示词`：`PromptAssembly`。
- `ENG:上下文`：`ExecutionContext` 携带单轮/短历史（内存）。
- `ENG:驾驭`：路由结构化日志。

## 4. 配置

- `application.yml`：公共配置。
- `application-dev.yml`：`spring.profiles.active=dev` 时启用 Ollama URL、模型名。

## 5. 数据

- 无持久化库；用户固定常量 `001` / `user`。
