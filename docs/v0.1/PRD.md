# v0.1 产品需求文档（PRD）

## 1. 背景与范围

交付 **史诗①** 的启动里程碑：可运行的 Java 服务端、**意图场景识别（三源路由）**、**八大行动模式**统一契约与注册表；**Ask**、**ReAct** 可对接本地 Ollama 演示；其余模式以 Stub 占位。  
**不在本版本范围**：完整记忆系统、技能工具生产实现、多用户（沿用单用户 `001` / `user`）。

## 2. 用户故事

| ID | 角色 | 故事 | 价值 |
|----|------|------|------|
| US-1 | 开发者 | 作为开发者，我希望启动 Spring Boot 后能通过健康检查确认服务就绪，以便联调与部署 | 可观测 |
| US-2 | 客户端/调用方 | 作为调用方，我希望能通过 HTTP 提交用户消息并可选指定行动模式，以便验证路由优先级 | 意图路由 |
| US-3 | 客户端/调用方 | 作为调用方，我希望在未指定模式时系统按规则或模型建议选用模式，以便减少手工选择 | 自动化 |
| US-4 | 开发者 | 作为开发者，我希望 Ask/ReAct 能调用本地 Ollama 默认模型返回回复，以便验证 LLM 集成 | 端到端 |
| US-5 | 运维 | 作为运维，我希望路由决策被记录到日志，以便审计与排障 | 驾驭工程 |

## 3. 验收标准

| 编号 | 标准（Given / When / Then） |
|------|------------------------------|
| AC-1 | Given 服务已配置 dev  profile，When 访问 Actuator 健康端点，Then 返回 UP |
| AC-2 | Given 请求体携带 `mode=ASK`，When 调用聊天 API，Then 路由来源为客户端显式且执行 Ask 处理器 |
| AC-3 | Given 请求未携带 mode 且用户文本匹配某条配置规则，When 调用聊天 API，Then 路由来源为规则且模式符合规则映射 |
| AC-4 | Given 请求未携带 mode 且无规则命中，When 调用聊天 API，Then 走模型路由分支（本版本可落地为「分类提示 + 解析」或文档约定的降级策略）并得到有效 `ActionMode` |
| AC-5 | Given 选用 Ask 或 ReAct，When 调用聊天 API，Then 响应体包含模型生成的 `reply` 文本（Ollama 可用时） |
| AC-6 | Given 选用非 Ask/ReAct 模式，When 调用聊天 API，Then 返回 Stub 约定信息且不抛未处理异常 |
| AC-7 | Given 任意一次路由，When 处理完成，Then 日志中包含路由来源、选用模式、用户 ID（固定 `001`） |
| AC-8 | Given 代码库检索，When 查找用户常量，Then 存在统一常量 `001` / `user` |

## 4. 非目标

- 不提供生产级 WebSocket 全量实现（可后续 v0.2）。
- 不实现向量记忆与 Qdrant。
