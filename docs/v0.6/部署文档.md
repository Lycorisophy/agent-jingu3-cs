# v0.6 部署文档 — 记忆系统

## 1. 环境与构建

与 [v0.2 部署文档](../v0.2/部署文档.md) 相同：JDK 17+、Maven。主配置 [application.yml](../../agent-jingu3-cs-server/src/main/resources/application.yml)。

**CI**：见仓库根 [`.github/workflows/ci.yml`](../../.github/workflows/ci.yml)。

## 2. 数据源与迁移

- **开发默认**：H2 内存库 + Flyway（`classpath:db/migration`，脚本源 [`docs/data/migration`](../../docs/data/migration)）。启动依次执行 **V1～V6**（含记忆表 V4/V5 与身份/技能表 V6）。
- **生产**：覆盖 `spring.datasource.*`；以 Flyway 迁移为准，勿用手工 `ddl-auto` 替代。
- **本地 MySQL + Redis + Milvus**：复制 [`application-local.yml.example`](../../agent-jingu3-cs-server/src/main/resources/application-local.yml.example) 为 `application-local.yml`，使用 `--spring.profiles.active=dev,local`。

## 3. 外部依赖（可选）

| 组件 | 用途 | 默认 |
|------|------|------|
| Ollama | 聊天与嵌入 | 默认开启；嵌入模型见下表 |
| Milvus | 记忆向量存储与检索 | `jingu3.milvus.enabled=false` |
| Redis | 记忆列表二级缓存 | `jingu3.redis.enabled=false` |
| Elasticsearch | 事件全文索引与检索（v0.6-C） | `jingu3.elasticsearch.enabled=false` |

## 4. 配置项摘要（jingu3）

| 键 | 说明 |
|----|------|
| `jingu3.memory.api-enabled` | 是否注册记忆 REST |
| `jingu3.memory.max-list-size` | 列表最大条数 |
| `jingu3.memory.injection-enabled` | 是否在对话前注入检索记忆（需 Milvus） |
| `jingu3.memory.embedding-model` | Ollama 嵌入模型名 |
| `jingu3.memory.embedding-dimension` | 0 表示首次探测 |
| `jingu3.memory.retrieval-top-k` | 向量检索条数 |
| `jingu3.redis.enabled` / `host` / `port` / `list-ttl-seconds` | Redis 与列表缓存 TTL |
| `jingu3.milvus.enabled` / `host` / `port` / `collection-name` | Milvus |
| `jingu3.elasticsearch.enabled` / `host` / `port` / `scheme` / `index-events` / `username` / `password` | Elasticsearch（事件 API） |
| `jingu3.ollama.base-url` | Ollama 服务地址 |

## 5. 验收操作建议

1. **仅 MySQL/H2**：`POST /api/v1/memory/entries` 写入 → `GET` 列表能查到。
2. **Redis**：启用 `jingu3.redis.enabled=true`，连续两次 `GET` 列表（可通过日志或调试观察缓存命中，以实现为准）。
3. **Milvus + 注入**：启用 `jingu3.milvus.enabled=true` 与 `jingu3.memory.injection-enabled=true`，写入至少一条记忆后发起 `POST /api/v1/chat`，确认提示侧包含参考记忆段落（或日志无异常降级）。
4. **Elasticsearch 事件**：启用 `jingu3.elasticsearch.enabled=true`，确保 ES 8.x 可达；启动时会用 classpath `elasticsearch/jingu3-events-index.json` 创建索引（无 IK，便于空集群）；`POST /api/v1/events` 写入 → `GET /api/v1/events/search?userId=001&q=关键词` 能命中。若 ES 暂时不可用：`GET .../search` 仍 200 且 `data` 为空；`POST` 返回 **503**、`code`=`AG_50301`。
