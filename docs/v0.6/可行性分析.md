# v0.6 可行性分析

## 1. 目标与边界

在 v0.3～v0.5 工具与横切能力稳定后，引入**结构化记忆元数据**、**可选 Redis 列表缓存**、**可选 Milvus + Ollama 嵌入与向量检索**，以及**对话前记忆注入**（特性开关）。单用户种子 `001` 不变；多账户与 JWT 在 **v1.0**。

**存储终局**以 [记忆知识系统数据库选型分析第二版.md](../设计/记忆知识系统数据库选型分析第二版.md) 与 [系统数据存储方案.md](../设计/系统数据存储方案.md) 为准；当前代码已落地 **M1～M4** 子集，**Elasticsearch / Neo4j / MinIO 全链路**列为 v0.6-C 或 v0.7 初（见 [里程碑切片.md](./里程碑切片.md)）。

## 2. 技术依赖

| 依赖 | 说明 |
|------|------|
| Spring Boot + Flyway + MyBatis-Plus | 已有；迁移源目录 [docs/data/migration](../../docs/data/migration) |
| Ollama | `POST /api/embeddings`；模型默认 `qwen3-embedding:8b` |
| Milvus | 可选；未启用时无向量写入与 ANN 检索 |
| Redis | 可选；未启用时列表无二级缓存 |
| MySQL / H2 | 开发默认 H2；生产覆盖数据源 |

## 3. 风险与缓解

| 风险 | 缓解 |
|------|------|
| 中间件不可用拖垮对话 | Milvus/Redis 失败路径降级：注入跳过、列表直查 DB；见 `MilvusMemoryRetrievalService` 日志告警 |
| 向量维度与模型不一致 | `jingu3.memory.embedding-dimension` 为 0 时首次探测；换模型须新集合并迁移（见 [milvus-collection-design.md](./milvus-collection-design.md)） |
| 范围膨胀（事件 ES、图数据库） | 与路线图 **v0.6-A/B/C** 分批交付；M2 起单独 PR |
| 与第一版选型文档表述冲突 | 以**第二版**与 [系统数据存储-物化清单与测试数据.md](../设计/系统数据存储-物化清单与测试数据.md) 为执行口径 |

## 4. 结论

**可行**。已实现路径（M1～M4）可在无 Redis/Milvus 的环境下保持与 v0.5 等价行为（开关默认关闭或组件禁用）。后续 **ES 事件索引、Neo4j、MinIO** 在基础设施就绪后按里程碑切片扩展。
