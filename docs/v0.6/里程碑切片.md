## 建议交付切片

### M1 — 第一个可合并增量（推荐）

- **目标**：在不绑定全部中间件的前提下，建立 **记忆域边界**：包结构、`MemoryService`（或等价门面）接口、**MySQL Flyway 迁移** 中的元数据表（如 `memory_entry` / `fact_metadata` 草案，名称以详细设计为准）。
- **验收**：应用启动 + 迁移成功；提供 **仅集成测试或手动可调** 的写入/按 `userId` 查询 API（可暂不接 `ChatService`）。
- **不包含**：Qdrant 全文语义检索、Neo4j。

### M2 — 事件写入与 MongoDB

- **目标**：按选型文档将 **Events** 落 **MongoDB**（或文档模型等价实现）；与 M1 元数据关联。
- **验收**：单测或集成测覆盖「写事件 → 按会话/用户列出」。

### M3 — Redis 与工作记忆

- **目标**：短期上下文或会话缓存（与现有 `conversationId` 对齐）；策略与 TTL 在详细设计中定稿。
- **验收**：开关关闭时零行为变化。

### M4 — 向量检索（Qdrant + 嵌入）

- **目标**：对接 Ollama 嵌入模型与 Qdrant；**检索注入** `ChatService` / `ChatStreamService`（Feature flag）。
- **验收**：AC-M3（PRD）在开启注入时可观测 prompt 或日志中的记忆片段（勿泄露密钥）。

### M5+ — P1/P2 组件

- Meilisearch / Neo4j / MinIO / InfluxDB 按产品优先级单列里程碑，避免与 M1～M4 同 PR 混杂。
