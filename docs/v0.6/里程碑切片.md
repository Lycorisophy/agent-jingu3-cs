# v0.6 里程碑切片 — 对齐存储终局（第二版）

> **2026-04-13 起**：以 [记忆知识系统数据库选型分析第二版.md](../设计/记忆知识系统数据库选型分析第二版.md) 与 [系统数据存储方案.md](../设计/系统数据存储方案.md) 为准；第一版选型中的 MongoDB / Meilisearch / InfluxDB 等**不作为**本里程碑默认组件。

## 建议交付切片

### M1 — 第一个可合并增量（推荐）

- **目标**：在不绑定全部中间件的前提下，建立 **记忆域边界**：包结构、`MemoryService`（或等价门面）接口、**MySQL Flyway 迁移** 中的元数据表（如 `memory_entry` / `fact_metadata` 草案，名称以详细设计为准）。
- **验收**：应用启动 + 迁移成功；提供 **仅集成测试或手动可调** 的写入/按 `userId` 查询 API（可暂不接 `ChatService`）。
- **不包含**：Qdrant 全文语义检索、Neo4j。

### M2 — 事件写入与 Elasticsearch

- **目标**：按第二版将 **Events** 落 **Elasticsearch**（索引如 `jingu3-events`）；与 `conversation_id` / `user_id` / `event_id` 对齐；映射与 Bulk 示例见 [物化清单](../设计/系统数据存储-物化清单与测试数据.md)。
- **验收**：单测或集成测覆盖「写事件 → 按用户/会话关键词检索」；可选异步管道，失败须可观测且不拖垮主对话路径。
- **已补充（实现）**：ES 检索异常时返回空列表 + WARN；写入失败 503（`AG_50301`）；与主对话链路仍隔离。

### M3 — Redis 与工作记忆

- **目标**：短期上下文或会话缓存（与现有 `conversationId` 对齐）；策略与 TTL 在详细设计中定稿。
- **验收**：开关关闭时零行为变化。
- **已落地（增量）**：`jingu3.redis.enabled` + `StringRedisTemplate`；`GET /api/v1/memory/entries` 列表 **可选** Redis 缓存（TTL `jingu3.redis.list-ttl-seconds`）；默认 `enabled=false` 不连 Redis。

### M4 — 向量检索（Milvus + Ollama 嵌入）

- **目标**：对接 Ollama `POST /api/embeddings` 与 **Milvus**（与选型报告中的 Qdrant 为同类能力，可二选一）；**检索注入** `ChatService` / `ChatStreamService` / `ModePlanExecutor`（`jingu3.memory.injection-enabled`）。
- **验收**：AC-M3（PRD）在开启注入且 Milvus/Ollama 可用时，模型输入前可带「参考记忆」段落；关闭注入或 `jingu3.milvus.enabled=false` 时与未接向量前行为一致。
- **Schema**：Flyway `V5__memory_embedding`；集合名 `jingu3.milvus.collection-name`（默认 `jingu3_memory`）。

### M5+ — Neo4j / MinIO 等

- **Neo4j**：事件关系、GraphRAG；与 ES `event_id` 对齐。  
- **MinIO**：附件与生成物；与技能包桶（v0.7）统一运维时可合并里程碑。  
- 按产品优先级单列 PR，避免与 M1～M4 同批混杂。
