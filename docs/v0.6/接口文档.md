# v0.6 接口文档 — 记忆实验 API

**基路径**：`/api/v1/memory`  
**统一响应**：`ApiResult<T>`（与全局约定一致）  
**开关**：`jingu3.memory.api-enabled`（默认 `true`）；为 `false` 时不暴露本组接口。

## 1. 创建记忆条目

- **POST** `/api/v1/memory/entries`
- **Content-Type**：`application/json`

**请求体**

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| userId | string | 是 | 用户 ID；单用户阶段常用 `001` |
| kind | string | 是 | `FACT` 或 `EVENT` |
| summary | string | 否 | 摘要 |
| body | string | 否 | 正文 |
| factTag | string | 否 | 仅 `kind=FACT` 时写入 `fact_metadata.tag` |

**响应**：`ApiResult<MemoryEntryVo>`

**MemoryEntryVo 字段**：`id`、`userId`、`kind`、`summary`、`body`、`factTag`、`createdAt`、`updatedAt`（ISO-8601 字符串，以实际序列化为准）。

## 2. 按用户列出记忆条目

- **GET** `/api/v1/memory/entries?userId={userId}`
- **参数**：`userId` 必填

**响应**：`ApiResult<List<MemoryEntryVo>>`；按创建时间倒序，条数上限 `jingu3.memory.max-list-size`。
