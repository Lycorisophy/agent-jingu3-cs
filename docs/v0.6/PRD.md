# v0.6 PRD — 记忆系统（初稿）

## 1. 背景与目标

在 v0.3～v0.5 工具与横切能力稳定后，引入 **事件记忆**、**事实记忆** 及 **时间维度策略**（永驻/长期/短期），并在对话链路中支持 **检索注入与窗口策略**，对齐行动模式设计指南与 [记忆知识系统数据库选型分析.md](../设计/记忆知识系统数据库选型分析.md)。

## 2. 用户故事（与路线图对齐）

1. **作为** 用户 **我希望** 重要事实被记住 **以便** 后续对话无需重复背景。
2. **作为** 用户 **我希望** 关键事件可追溯 **以便** 审计与续聊一致。
3. **作为** 产品 **我希望** 记忆写入与检索可配置 **以便** 控制成本与隐私。

## 3. 范围（路线图功能需求摘要）

- 事件记忆：模型、存储、检索 API
- 事实记忆：模型、存储、检索 API
- 时间维度：永驻、长期、短期行为定义与实现
- 上下文：检索注入与窗口策略
- 嵌入与向量存储：Ollama 嵌入 + **Milvus**（与选型文档中的 Qdrant 为同类能力，仓库默认 Milvus）分阶段落地
- 关系库：元数据与会话等结构化数据

## 4. 验收标准（初稿）

| 编号 | 验收项 |
|------|--------|
| AC-M1 | 完成 [里程碑切片.md](./里程碑切片.md) 中 **M1** 定义的可运行增量（元数据表 + 写入/查询 API 或等价路径） |
| AC-M2 | 事件/事实至少一类可走通「写入 → 持久化 → 按用户/会话检索」 |
| AC-M3 | 对话主链路可注入记忆片段（配置开关关闭时行为与 v0.5 一致） |
| AC-M4 | 部署文档说明依赖的外部存储（MySQL / Redis（可选）/ Milvus（可选）/ MongoDB 等按实际采纳项） |
