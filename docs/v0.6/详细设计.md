# v0.6 详细设计

## 1. 数据库

### 1.1 表（Flyway）

| 脚本 | 表 | 说明 |
|------|-----|------|
| V4 | `memory_entry` | `id` BIGINT PK；`user_id`、`kind`（FACT/EVENT）、`summary`、`body`、时间戳 |
| V4 | `fact_metadata` | `memory_entry_id` PK/FK；`tag` |
| V5 | `memory_embedding` | `memory_entry_id` PK/FK；已向量化标记 |
| V6 | `users`、`conversations`、`messages` 等 | 与记忆 API 无直接外键；对话持久化为后续增量 |

### 1.2 枚举

- `MemoryEntryKind`：`FACT`、`EVENT`（与 `CreateMemoryEntryRequest.kind` 一致，大写）。

## 2. REST API

见 [接口文档.md](./接口文档.md)。控制器：[MemoryController.java](../../agent-jingu3-cs-server/src/main/java/cn/lysoy/jingu3/controller/MemoryController.java)。

- `jingu3.memory.api-enabled=false` 时不注册 `MemoryController`。
- **更新**：`PUT /api/v1/memory/entries/{id}`，`DefaultMemoryService#update` 写库后调用 `MemoryVectorIndexer#afterMemoryUpdated`（Milvus 实现为先 `deleteByMemoryEntryId`、删 `memory_embedding` 行，再按创建逻辑重嵌入）。
- **删除**：`DELETE ...?userId=`，`onMemoryDeleted` 仅删 Milvus 向量；随后 `memory_entry` 删除，`memory_embedding` / `fact_metadata` 由外键级联删除。

## 3. 写入与向量索引

**创建条目**（`DefaultMemoryService#create`）：

1. 插入 `memory_entry`。
2. 若 `kind=FACT` 且提供 `factTag`，插入 `fact_metadata`。
3. 失效该用户列表缓存（Redis）。
4. 调用 `MemoryVectorIndexer.afterMemoryCreated`：在 Milvus 启用且嵌入可用时写入向量并插入/更新 `memory_embedding`。

**列表**（`listByUserId`）：先读 Redis 缓存（若启用）；未命中则查库，上限 `jingu3.memory.max-list-size`。

## 4. 对话注入

- 入口：`MemoryAugmentationService#augmentUserMessageIfEnabled`，由 `ModePlanExecutor`、`ChatService`、`ChatStreamService` 在构造执行上下文前调用（以代码为准）。
- 实现类：`MilvusMemoryRetrievalService`（`@ConditionalOnProperty` `jingu3.milvus.enabled=true`）。
- 逻辑：对用户当前句做嵌入 → Milvus `searchSimilar(userId, queryVector, topK)` → 按返回 `memory_entry_id` 拉取行 → 拼接 `PromptFragments.MEMORY_REFERENCE_HEADER` 与条目摘要/正文预览 → 前缀到用户消息前。
- 失败：捕获异常，打日志，返回**原始用户消息**。

## 5. Milvus

- 客户端封装：`MilvusMemoryVectorService`；集合名 `jingu3.milvus.collection-name`（默认 `jingu3_memory`）。
- Schema、索引、过滤表达式：见 [milvus-collection-design.md](./milvus-collection-design.md)。

## 5.1 Elasticsearch（v0.6-C）

- **开关**：`jingu3.elasticsearch.enabled`（默认 `false`）。
- **客户端**：`co.elastic.clients.elasticsearch.ElasticsearchClient`，配置类 `cn.lysoy.jingu3.events.config.ElasticsearchClientConfiguration`。
- **索引**：启动时 `EventIndexInitializer` 若索引不存在则根据 `classpath:elasticsearch/jingu3-events-index.json` 创建（**标准分词**，生产可换为带 IK 的映射，见 [物化清单](../设计/系统数据存储-物化清单与测试数据.md)）。
- **API**：`EventDocumentController` — `POST /api/v1/events` 写入，`GET /api/v1/events/search` 多字段检索；实现 `DefaultEventIndexingService`。
- **与对话主链路**：当前为**独立实验 API**；对话侧异步写 ES 可在后续里程碑接入。
