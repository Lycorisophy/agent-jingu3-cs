**文档约定**：每个版本的过程文档位于 **`docs/{版本号}/`**；工序与清单见 [docs/v0.1/README.md](../v0.1/README.md) 模板（各版本可复制结构）。**补充设计**（不限于某一 v0.x 目录）见 `docs/workflow/`、`docs/workspace/` 等子目录；其中 [workflow-design.md](../workflow/workflow-design.md) 为 **BPMN/Flowable 扩展愿景**，与下述 v0.4「JSON 工作流」里程碑区分；[workspace-design.md](../workspace/workspace-design.md) 为 **工作空间（安全文件 + 代码沙箱 + 工具注册）** 的已定稿设计；**技能市场、OSS 与客户端执行** 见 [skill-system-design.md](../workspace/skill-system-design.md)。**实现里程碑见 v0.7**。

**单用户（用户系统完成前）**：用户 ID `001`，用户名 `user`。

**客户端（独立仓库）**：每轮响应中的 **`actionMode` / `routingSource`**（非流式见 `ChatVo`，流式见首包 `StreamEvent` `META`）为「当前行动模式」的**唯一事实来源**；`routingSource` 含 **`EXPLICIT_GUARD`**（显式 Plan/Team 经守门降为 Ask，见 v0.5 进展）。UI 展示、编排逐步模式、`TOOL_RESULT` 等与后端对齐约定见 [JinGu3-AI-Agent客户端UI设计规范.md](../设计/JinGu3-AI-Agent客户端UI设计规范.md) **§1.4、§5.6**。

## v0.5 — Cron + HITL + State 持久化 + Agent Team 增强边界（差距整改 · 横切）

**版本状态**：**本批次功能已落地**（2026-04-12）：B1 守门、Cron MVP、**HITL**、**DST（`dialogue_state`）**、**Agent Team 多轮+合成**；文档工序见 `docs/v0.5/` 勾选；发布前可按 PRD 做一轮验收与回归。

**目标**（对齐差距分析 §3.5–3.8）：**Cron** 任务持久化与调度触发；**HITL** 审批队列与审批 API；**State** 任务级状态表（与 conversation 关联）；**Agent Team** 在可控范围内增强（多轮/多角色消息结构，见 `docs/v0.5/详细设计.md`）。与 [Cron与HITL模块设计草案.md](../设计/Cron与HITL模块设计草案.md) 对齐。

**进展（2026-04-12）**

- **守门（B1）**：`IntentRouter` 在显式 `PLAN_AND_EXECUTE` / `AGENT_TEAM` 下调用 `ModelIntentClassifier#classifyOptional`；分类为 ASK/REACT 时实际执行 **Ask**，`RoutingSource` 为 **`EXPLICIT_GUARD`**，`RoutingDecision#guardUserNotice` 为用户可见说明；分类器调用失败则尊重显式模式。非流式：`reply` 前缀拼接；流式：`META` 后先发 **`BLOCK`（说明）**，再进入 Ask `TOKEN` 流。详见 [v0.5-横切能力极简设计.md §1](../设计/v0.5-横切能力极简设计.md)。
- **配置**：`jingu3.routing.explicit-mode-guard-enabled`（默认 `true`）；`false` 时显式 Plan/Team **直通** Handler，不调用 `classifyOptional`（与 v0.4 行为一致）。
- **流式占位模式**：`CronModeHandler` / `StateTrackingModeHandler` / `HumanInLoopModeHandler` 实现 `stream`（`STEP_BEGIN` + `BLOCK` + `DONE`）。
- **提示词前缀**：`ModeRoutingPreamble` 覆盖 `RoutingSource.EXPLICIT_GUARD`。
- **设计草案**：同文档含阶段 C 决议（ReAct/Workflow 暂不同等守门；`cardKind`：`state_digest` / `action_approval`）及 B2/B3/B4 极简表字段与 REST 占位。
- **排期索引**：[docs/v0.5/README.md](../v0.5/README.md) 列出下一批建议顺序（HITL → DST → Agent Team）。
- **Cron（MVP，2026-04-12）**：`db/migration/V1__scheduled_task.sql`；`ScheduledTaskPoller` + `ScheduledTaskDueRunner`（`CronChatBridge` 隔离对话事务）；`CronController` **`/api/v1/cron/tasks`**；默认 H2 内存库（生产请覆盖 `spring.datasource`）；MVP 为**单次执行**（执行后 `enabled=false`），`cron_expression` 仅预留展示。
- **HITL（MVP）**：`V2__hitl_approval.sql`；`HitlController` **`/api/v1/hitl`**（创建、`pending`、approve/reject）。
- **DST（占位）**：`V3__dialogue_state.sql`；`DialogueStateController` **`/api/v1/dst/{conversationId}`**（GET/PATCH/confirm）。
- **Agent Team**：`AgentTeamModeHandler` 配置化多轮 Specialist + 合成步；流式 `specialist_{k}` / `synthesize`。

**与 v0.8 的关系**：设计稿以 `userId` 区分工作空间；在 **单用户种子 `001`** 阶段可先实现「每进程/每配置根目录」的 MVP，**多用户物理隔离与配额强约束**随 v0.8 鉴权与用户体系一并验收。

| 阶段 | 内容摘要 |
|------|----------|
| **Phase 1** | `WorkspaceManager`、`WorkspaceFileService`、路径安全（如 `PathValidator`）、基础工具注册 |

| **Phase 3** | 执行历史（`execution_history` 等）、存储配额、`jingu3.workspace.*` / `jingu3.sandbox.*` 配置、管理/调试 REST（路径实现时统一为 **`/api/v1/...`**） |
