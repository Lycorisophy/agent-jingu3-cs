# jingu3 CS — 开发路线图

| 版本 | 日期 | 说明 |
|------|------|------|
| 1.0 | 2026-04-12 | 首版：史诗、v0.1+ 版本目标、文档工序勾选 |
| 2.0 | 2026-04-12 | **差距整改重排**：新 v0.3～v0.5 对齐 [行动模式实现差距分析.md](../设计/行动模式实现差距分析.md)；原记忆/技能/用户史诗顺延至 v0.6～v0.8 |

**史诗顺序（v2.0 起）**：① 意图场景识别 + 八大行动模式（v0.1～v0.2 已完成）→ **② 工具真能力 + 编排增强 + 横切能力（新 v0.3～v0.5）** → ③ 记忆系统（v0.6）→ ④ 技能系统扩展（v0.7）→ ⑤ 用户系统（v0.8）。

**与差距分析的关系**：报告结论认为 **Tool Registry** 为 P0 根因；故将「工具子系统 MVP」前移到 **v0.3**，原路线图中的「v0.3 记忆」整体后移至 **v0.6**，避免与指南 §3/§4 对齐冲突。详见各版本节内说明。

**文档约定**：每个版本的过程文档位于 **`docs/{版本号}/`**；工序与清单见 [docs/v0.1/README.md](../v0.1/README.md) 模板（各版本可复制结构）。

**本地 Ollama**：主模型 `gnremy/qwen3.5-abliterated:35b-a3b`，嵌入 `qwen3-embedding:8b`。

**单用户（用户系统完成前）**：用户 ID `001`，用户名 `user`。

## 全局 — 文档工序（每个 v0.x 发布前完成）

适用于每一个版本目录（如 `docs/v0.1/`）：

- [ ] 1. PRD（含用户故事、验收标准）— `PRD.md`
- [ ] 2. 可行性分析 — `可行性分析.md`
- [ ] 3. 概要设计 — `概要设计.md`
- [ ] 4. 详细设计 — `详细设计.md`
- [ ] 5. 服务端实现（含注释）— 代码在 `agent-jingu3-cs-server/`
- [ ] 6. 接口文档 — `接口文档.md` 或 `openapi.yaml`
- [ ] 7. 前端实现（若本版本包含）— 客户端工程
- [ ] 8. 后端单元测试
- [ ] 9. 部署文档 — `部署文档.md`

## v0.1 — 引擎内核与路由契约（史诗① 启动）

**版本状态**：已完成（2026-04-12）

**目标**：Spring Boot 工程可运行；**意图场景识别**三源路由与优先级；**ModeRegistry** 与八模式统一契约；Ask/ReAct 等至少部分真跑通；最小会话上下文（**不计入**记忆系统史诗完成）；用户常量 `001`/`user`。

### 文档与工序（`docs/v0.1/`）

- [x] PRD → 可行性 → 概要 → 详细 → 服务端代码 → 接口文档 → 前端（本版本跳过）→ 单测 → 部署文档

### 功能需求

- [x] `SYS:意图路由` 三源路由接口：客户端显式 > 规则 > 模型
- [x] `SYS:行动模式` ModeRegistry + 八模式接口；至少 Ask、ReAct 可演示
- [x] `SYS:行动模式` 其余模式 Stub 或占位返回（接口齐全）
- [x] `ENG:提示词` 基础提示词模板与拼装
- [x] `ENG:上下文` 引擎所需最小上下文（非记忆史诗）
- [x] `ENG:驾驭` 路由决策日志/可观测
- [x] Spring Boot：替换占位 Main、健康检查、配置分层（如 dev/prod）
- [x] 单用户 `001` / `user` 常量或配置

**本版本关闭条件**：上列全部勾选；`docs/v0.1/` 文档工序齐全。

## v0.3 — 工具子系统 MVP + Ask / ReAct 真闭环（差距整改 · Phase 1）

**版本状态**：未开始

**目标**（对齐 [行动模式实现差距分析.md](../设计/行动模式实现差距分析.md) §3.1–3.2、§5 Phase 1）：落地 **Tool Registry**（或 LangChain4j Tool 薄封装）、工具发现与执行管线；**Ask** 支持「意图 → 选工具或直答 → 汇总」；**ReAct** 每步 **Action = 真实工具调用**，**Observation = 工具结果** 写回上下文（替代纯提示词模拟）。至少 1～2 个可验证内置工具（如计算器、占位检索）。

**依赖**：无前置史诗；基于现有 `agent-jingu3-cs-server` 与 Ollama。

- [ ] `SYS:技能` 工具注册表与单次执行（MVP）
- [ ] `SYS:行动模式` AskModeHandler / ReActModeHandler 接入真实工具路径
- [ ] `ENG:提示词` 工具描述与路由提示（与指南 §3 对齐）
- [ ] `ENG:驾驭` 工具调用日志与错误封装
