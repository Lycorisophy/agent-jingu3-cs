# v0.3 接口文档

在 [v0.2 接口文档](../v0.2/接口文档.md) 基础上，**HTTP 路径与 `ChatRequest` / `ChatVo` 不变**；本版补充 **流式事件 `TOOL_RESULT`** 与 **配置项**。

## 1. 配置项（`application.yml`）

| 键 | 类型 | 默认 | 说明 |
|----|------|------|------|
| `jingu3.tool.enabled` | boolean | `true` | 为 `false` 时关闭 Ask 路由与 ReAct 工具执行，行为对齐 v0.2 |

## 2. 流式 `StreamEvent` 扩展

每条事件为 JSON，字段随 `type` 选填（与 v0.1/v0.2 一致）。

### 2.1 新增类型 `TOOL_RESULT`

在 **Ask** 或 **ReAct** 流式路径中，工具同步执行完成后、最终 `TOKEN` 流之前，可下发：

| 字段 | 类型 | 说明 |
|------|------|------|
| type | string | 固定 `TOOL_RESULT` |
| toolId | string | 注册表工具 id，如 `calculator`、`utc_time` |
| toolOutput | string | 工具返回文本 |

客户端可据此展示「工具卡片」或调试信息；未实现专用 UI 时可忽略。

### 2.2 事件顺序（Ask + 工具）

1. `META`
2. （阻塞）内部完成路由 LLM 调用
3. 若走工具：`TOOL_RESULT`
4. `TOKEN`…（汇总阶段）
5. `DONE`

## 3. 内置工具 ID（供联调）

| toolId | 行为 |
|--------|------|
| `calculator` | 四则表达式求值，`input` 为表达式字符串 |
| `utc_time` | 返回当前 UTC 的 ISO-8601 文本，`input` 可空 |
