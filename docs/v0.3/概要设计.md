# v0.3 概要设计

## 1. 模块边界

| 模块 | 职责 |
|------|------|
| `tool`（新建，包名实现时确定） | 注册表、执行器、内置工具 |
| `engine/mode` | Ask、ReAct 接入工具管线 |
| `prompt` | 工具列表注入、ReAct 观察段模板 |

## 2. 数据流（Ask）

用户消息 →（可选）工具路由决策 → 工具执行或跳过 → LLM 汇总 → 返回用户。

## 3. 数据流（ReAct）

循环：Prompt（含历史 Observation）→ LLM 步输出 → 解析工具调用 → 执行 → 结果写入 Observation → 直至终止。

## 4. 非功能

- 日志：工具名、耗时、成功/失败（脱敏）。
- 配置：`application.yml` 中可开关「是否启用工具」便于回滚。
