# v0.3 可行性分析

## 1. 技术可行性

- **LangChain4j + Ollama**：项目已集成 `langchain4j-ollama`；工具调用需核对所用模型是否支持原生 tool calling。若不支持，采用 PRD/详细设计中的「两轮 JSON 协议」降级方案，工程上可行。
- **与现有引擎兼容**：在 `ModeRegistry` 不增新模式的前提下替换 `AskModeHandler`/`ReActModeHandler` 内部实现，对外 API 不变。

## 2. 风险

| 风险 | 缓解 |
|------|------|
| 模型不按格式输出工具 JSON | 强提示词 + 重试一次 + 回退直答 |
| 工具滥用 | v0.3 仅开放低危内置工具；鉴权与分级在 v0.7 强化 |

## 3. 结论

建议按 v0.3 范围实施。定稿采用 **两轮 JSON 协议**（Ask 路由 + 汇总；ReAct 页脚 `<<<JINGU3_JSON>>>`），不依赖 Ollama 原生 tool calling，降低模型差异风险。若后续模型与 LangChain4j 对 tool calling 支持稳定，可在注册表之上增加适配层，无需改变对外 API。
