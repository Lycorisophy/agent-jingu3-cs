# v0.3 详细设计 — 工具子系统与 Ask / ReAct 集成（定稿）

## 1. 对照指南

| 指南章节 | 本设计落点 |
|----------|------------|
| §3 Ask | 一轮「路由 JSON」→（可选）工具执行 → 第二轮「汇总」生成 |
| §4 ReAct | Thought → Action（JSON `invoke`）→ Observation（工具 stdout 写入「已有过程」）→ 循环直至 `done` / `[TASK_COMPLETE]` / 步数上限 |

## 2. 包与组件

| 组件 | 包 / 类 | 说明 |
|------|---------|------|
| 工具契约 | `cn.lysoy.jingu3.tool.Jingu3Tool` | `id()` / `description()` / `execute(String)` |
| 注册表 | `cn.lysoy.jingu3.tool.ToolRegistry` | 注入全部 `Jingu3Tool` Bean；`execute(id,input)`；`buildCatalogMarkdown()` |
| 内置工具 | `CalculatorTool`（`calculator`）、`UtcTimeTool`（`utc_time`） | 低危、可单测 |

| 配置 | `Jingu3Properties.Tool` | `jingu3.tool.enabled`，默认 `true` |

## 3. JSON 协议

### 3.1 Ask 工具路由（首轮 `generate` 输出应仅一行 JSON）

- 直答：`{"route":"direct"}`
- 调工具：`{"route":"tool","toolId":"<id>","input":"<单字符串>"}`

解析失败或空结果：**回落**为直答（第二次 `generate` 使用 `buildAskCombinedPrompt`）。

### 3.2 ReAct 页脚（模型输出末尾）

1. 推荐：单独一行 `<<<JINGU3_JSON>>>`，下一行 JSON。
2. 兼容：从全文提取**最后一个**平衡括号 JSON 对象。

载荷：

- 调用：`{"action":"invoke","toolId":"<id>","input":"<参数>"}`
- 结束：`{"action":"done"}`

正文仍可使用 `[TASK_COMPLETE]` 作为补充结束标记。

## 4. 流式事件（SSE / WebSocket）

| `StreamEventType` | 含义 | 主要字段 |
|-------------------|------|----------|
| `TOOL_RESULT`（v0.3） | 工具已执行 | `toolId`、`toolOutput` |

Ask 流式：先发路由阻塞 `generate`，再（若需要）`toolResult`，最后对汇总提示词走 `TOKEN` 流。

## 5. 依赖

- **exp4j**：`calculator` 表达式求值；字符白名单防注入。

## 6. 错误处理

- 未知工具 / 执行异常：`ToolExecutionException`；Ask 将错误注入 `buildAskToolFailurePrompt`；ReAct 将 `[系统观察] 工具错误：…` 写入「已有过程」。
