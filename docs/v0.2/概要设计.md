# v0.2 概要设计

## 1. 系统边界

```
[HTTP Client] --> [ChatController] --> [ChatService]     # 同步 JSON
                      |
                      +--> [ChatStreamService]  # 流式（SSE / WebSocket 共用事件）
                      |
         +------------+-------------+
         |                          |
   [ModePlanExecutor]          [IntentRouter] --> [ModeRegistry] --> [ActionModeHandler]
         |                          |
         +------------+-------------+
                        |
                  [PromptAssembly / 常量]
```

- **同步**：`POST /api/v1/chat` → `ApiResult<ChatVo>`（与 v0.1 兼容）。
- **流式**：`POST /api/v1/chat/stream`（`Content-Type: text/event-stream`），每条 SSE `data` 为与 WebSocket 相同的 `StreamEvent` JSON；**WebSocket** `ws://<host>/ws/v1/chat`，连接后首条文本消息为 `ChatRequest` JSON。`ChatStreamService` 与 `ChatService` 共用校验与路由；**ASK** 模式使用 Ollama 流式 API 推送 `TOKEN`；多步模式（ReAct、Plan、Workflow、Agent Team 等）推送 `STEP_BEGIN` / `BLOCK` / `STEP_END`。详见下文「流式事件」。

- 若请求含 **非空 `modePlan`**：不走三源路由选单模式，由 **ModePlanExecutor** 顺序执行；路由来源记为 **CLIENT_EXPLICIT**，`note` 含 `modePlan`。
- 否则：沿用 v0.1 **IntentRouter**（客户端单 `mode` > 规则 > 模型）；非法单 `mode` → **FALLBACK** + `REACT`。
- 与《行动模式设计指南》章节级对照见 [行动模式指南-实现对照表.md](../设计/行动模式指南-实现对照表.md)。

## 2. 模块划分

| 模块 | 职责 |
|------|------|
| `controller` | `ChatController`：同步 `/chat` + SSE `/chat/stream` |
| `service` | `ChatService`：同步编排；`ChatStreamService`：流式编排（`StreamEvent`） |
| `engine.orchestration` | `ModePlanExecutor`：解析、降级、步间拼接、聚合 `planSteps` |
| `engine.routing` | `IntentRouter` 扩展非法 mode 降级；`RoutingSource` 增加 `FALLBACK` |
| `engine.mode` | 各模式独立 `ActionModeHandler` Bean；**移除**对六模式的统一 Stub 依赖（Stub 可删除或仅作兜底） |
| `common.dto` / `common.vo` | `ChatRequest.modePlan`；`ChatVo.planSteps`（`PlanStepVo` 列表） |

## 3. 与三大系统/工程映射

- `SYS:行动模式` + `SYS:意图路由`：编排 + 单模式路由与降级。
- `ENG:提示词`：各新模式 handler 使用 `PromptAssembly` / `PromptTemplates`。
- `ENG:驾驭`：日志含 `routingSource`、`note`、编排步摘要。

## 4. 数据

- 无持久化；**State Tracking** 使用 `conversationId` 为键的 **进程内** `ConcurrentHashMap`（重启丢失，v0.3 可换存储）。

## 5. 流式事件（SSE 与 WebSocket 一致）

| `type` | 含义 |
|--------|------|
| `META` | 路由结果：`actionMode`、`routingSource`、`userId`、`username` |
| `TOKEN` | 增量文本（ASK 打字机） |
| `STEP_BEGIN` / `STEP_END` | 多步边界：`step`、`label` |
| `BLOCK` | 阻塞式一整段（多步子调用或非流式模式） |
| `DONE` | 正常结束 |
| `ERROR` | 失败信息：`error` |

**示例（curl SSE）**：`curl -N -H "Content-Type: application/json" -d "{\"message\":\"你好\"}" http://localhost:8080/api/v1/chat/stream`

**WebSocket**：浏览器 `new WebSocket('ws://localhost:8080/ws/v1/chat')`，`open` 后 `send(JSON.stringify({ message: '你好', mode: 'ASK' }))`，逐帧解析 JSON 行即可复用与 SSE 相同的前端拼接逻辑。
