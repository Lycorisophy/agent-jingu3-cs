# 代码质量与扫描（Alibaba / SonarLint / Qodana）

本仓库服务端代码约定对齐 **《阿里巴巴 Java 开发手册》**（通过 **P3C PMD** 规则集）、本地 **SpotBugs**（与 Sonar 部分 Bug 规则重叠），以及 JetBrains **Qodana**（与 IDE 检查、Qodana 云端一致）。

## 1. 职责划分

| 工具 | 作用 |
|------|------|
| **P3C PMD**（Maven） | 阿里巴巴手册：命名、常量、并发、异常、OOP 等 |
| **SpotBugs**（Maven） | 潜在 Bug、空指针、错误用法 |
| **SonarLint**（IDE） | 在 Cursor/IDEA 中实时提示；规则与 SonarQube 同源思路，提交前处理告警 |
| **Qodana**（Docker/CI） | JVM 全量检查、报告与门禁，配置见仓库根目录 [qodana.yaml](../../qodana.yaml) |

## 2. 本地一键扫描（Maven）

在**仓库根**（推荐）或 **`agent-jingu3-cs-server`** 目录执行：

```bash
mvn -q clean verify
```

（仓库根存在父 `pom.xml` 聚合子模块；与 CI 一致。）

将执行：编译、测试、**PMD（p3c）**、**SpotBugs**。若有违规，`verify` 失败。

仅跑静态分析（跳过测试可加 `-DskipTests`）：

```bash
mvn -q clean compile pmd:check spotbugs:check
```

报告位置：

- PMD：`target/pmd.xml`
- SpotBugs：`target/spotbugsXml.xml`

## 3. Qodana（需 Docker）

在仓库根目录：

```bash
docker run --rm -v "%CD%:/data/project" -p 8080:8080 jetbrains/qodana-jvm:latest --show-report
```

浏览器打开提示的本地报告地址。

## 4. SonarLint

在编辑器安装 **SonarLint** 插件，绑定本仓库；与 `mvn verify` 互补，优先修复 **阻断 / 严重** 级别问题后再提交。
